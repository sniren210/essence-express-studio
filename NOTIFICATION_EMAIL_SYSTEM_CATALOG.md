# Daftar Notifikasi Email dan Aplikasi Creelab

Dokumen ini dibuat untuk pembaca non-teknis. Isinya menjelaskan jenis notifikasi yang bisa diterima pengguna Creelab, baik lewat email maupun notifikasi di aplikasi.

## Cara Membaca

| Istilah | Artinya |
| --- | --- |
| Email | Pesan dikirim ke alamat email pengguna. |
| Notifikasi aplikasi | Pesan muncul di aplikasi Creelab, misalnya di halaman notifikasi atau badge unread. |
| Admin | Tim internal Creelab yang meninjau laporan, verifikasi, refund, dispute, dan proses lain. |
| Brand | Pihak yang membuat campaign, lowongan, atau order. |
| Kreator | Pihak yang menerima pekerjaan, mengerjakan order, dan menerima pendapatan. |
| Buyer/User | Pengguna yang memesan layanan atau terlibat di order. |

Catatan penting:

- Tidak semua pengguna pasti menerima email. Pengguna bisa mematikan email atau notifikasi aplikasi dari pengaturan.
- Beberapa notifikasi hanya untuk aplikasi, beberapa hanya email, dan banyak yang bisa dua-duanya.
- Judul dan isi notifikasi bisa berubah sesuai campaign, order, brand, kreator, alasan penolakan, nominal pembayaran, atau status terbaru.

| Notifikasi | Kenapa penting |

| Mediasi pembatalan order | Admin perlu membantu menyelesaikan konflik order. |
| Verifikasi brand diminta | Admin perlu meninjau dokumen brand. |
| Laporan baru atau status laporan berubah | Admin perlu memantau laporan pengguna. |
| Dispute pembayaran dibuat | Admin perlu mengecek sengketa pembayaran. |
| Refund menunggu proses | Admin perlu memastikan refund diproses. |
| Dana escrow ditahan | Admin perlu memantau alasan penahanan dana. |
| Admin perlu bertindak | Ada aksi manual yang perlu dilakukan tim internal. |

## Catatan Untuk Tim Produk, CS, dan QA

- Kalau pengguna bertanya "kenapa saya tidak menerima email?", cek pengaturan notifikasi pengguna dan status email service.
- Kalau pengguna bertanya "kenapa tidak muncul di aplikasi?", cek apakah tipe notifikasi dikirim sebagai notifikasi aplikasi dan apakah user mematikan notifikasi.
- Kalau isi email berbeda dari notifikasi aplikasi, itu normal karena email memakai template, sedangkan aplikasi memakai judul dan pesan pendek.
- Kalau notifikasi berhubungan dengan order, biasanya detailnya bisa dilihat lewat halaman order/campaign terkait.
- Kalau notifikasi berhubungan dengan report, refund, dispute, atau withdrawal, biasanya butuh tindak lanjut admin.
